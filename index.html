1.0 Instalation

1.2 Project

npm install --save-dev conjurate 

{
  "scripts": {
    "gen": "conjurate"
  }
}

1.3 No installation

npx conjurate

2.0 Getting start

2.1 Create the files of your template using the 5.0 template's placeholder

The name "component" must be the same used in the config file in 2.3

├── conjurate
    │   └── component
    │       ├── %kebab%.js
    │       └── %kebab%.css
    ├── components
    ├── package-lock.json
    └── package.json
    
./conjurate/component/%kebab%.js

    import React from 'react';
    import %camel%Style from './%kebab%.css'
2.2 Run conjurate initial command to genrate a config file or add the config to package.json
conjurate --init
    
├── conjurate
    │   └── component
    │       ├── %kebab%.js
    │       └── %kebab%.css
    ├── components
    ├── package-lock.json
    +├── .conjurate.json
    └── package.json
.conjurate.json 

{
  "templatesRoot": "./conjurate",
  "templates": {
    "<template-name>": "./<default-destination-dir>",
    "<example-component>": "<example-./src/components>"
  }
}
2.3 Change templates name and default destination folder

  {
    "templatesRoot": "./conjurate",
    "templates": {
    -  "template-name": "./template-destination-dir"
    -  "<example-component>": "<example-./src/components>"
    +  "component": "./components"
    }
  }
2.4 Run conjurate passing the template-name and the placeholder name
conjurate component paper-input

    ✔  success  created ./components/paper-input
    
├── conjurate
    │   └── components
    │       ├── %kebab%.js
    │       └── %kebab%.css
    ├── components
    │   └── input
    +│       ├── paper-input.scss
    +│       └── paper-input.js
    ├── package-lock.json
    └── package.json
    

    +  ./conjurate/input/index.js
    +
    +  import React from 'react';
    +  import paperInputStyle from './paper-input.css'

3.0 Usage

conjurate <template-name> <placeholder-name> [--out ./output]

3.1

The template name configured in 2.3.

3.2

It will be the generated folder name inside destination folder configured in 2.3.

3.3 --out` overwrite conjurate config file.

4.0 Config

{
      "templatesRoot": "./conjurate",
      "templates": {
        "template-name": "./default-destination-dir"
      }
    }

4.1 templatesRoot

This is the folder where you keep your templates.

├── conjurate
    ├── src
    └── package.json

 

4.2 templates

Templates have a name and a default destination directory.

The name of the folder is the same used in the CLI when you run the generator.

├── conjurate
    │   └── template-name
    │       ├── %camel%.js
    │       └── %kebab%.scss
    ├── src
    └── package.json
    
$ conjurate template-name placeholder-name [--out ./replace/destination/placeholder-dir]

 

5.0 Templates

You can use a placeholder when developing your templates to help with customization. It will be replaced by the and can be used in files names and in file content. (directory name will be allowed in next versions). In our example it was paper-input.

In the example we used %camel% and %kebab% but you have more options. They are inherited from change-chase package, eg:

conjurate component "test string"

If you want to use space in the placeholder name use string.

%camel%: => testString

%constant% => TEST_STRING

%lower% => test_string

%lcFirst% => tEST

%no%do nothing

%kebab% => test-case

%pascal% => TestString

%path% => test/string

%sentence% => Test string

%snake% => test_string

%swap%=> TEST STRING

%title% => Test String

%upper%=> TEST STRING

%ucFirst => Test string
